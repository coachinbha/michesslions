Upload Pairings
How it works
  1. Export pairings from SwissSys as a JSON file using the Online Pairings export.
  2. Load your key file to autofill your GitHub details. You only make this once.
  3. Drop in the SwissSys export and hit Push to GitHub.
  4. GitHub Pages updates in about 30 seconds. The pairings page refreshes automatically.
▶ How to create your key file

Your key file is a .json file saved on your computer. It holds your GitHub details so you don't type them every time. Keep it private. Treat it like a password.


1. Create a GitHub fine-grained PAT
Go to github.com/settings/tokens and click Generate new token (fine-grained).
Set repository access to just your pairings repo.
Under permissions, set Contents to Read and write. Nothing else needed.


2. Create the file
Make a file named chess-key.json (or anything you like) with this content:

{
  "pat":   "github_pat_xxxxxxxxxxxx",
  "owner": "your-github-username",
  "repo":  "your-repo-name",
  "path":  "pairings.json"
}

path is where the file saves in your repo. pairings.json in the root works if your site fetches ./pairings.json.
